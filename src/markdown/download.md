## Why a Daily Build?

### Current Moonbase

Daily images are built from the current state of Lunar's module collection.

### Jenkins CI

The ISO is generated automatically by the Lunar Linux build infrastructure.

### Fresh starting point

New installations start closer to today's system instead of old media.

## Before you install

Lunar Linux is primarily source-based. Most software is compiled locally on the target machine.

Installation and upgrades may take longer than on binary distributions, but the result is a system built with the administrator's choices.

Expect a hands-on installation process, optional dependency decisions, and a system that remains close to traditional Linux administration.

> Start current. Build only what you need.  
> Understand every part of your system.

## What comes next?

### Build from source

Most modules are compiled locally on the machine where they will run.

### Optional dependencies

Choose what functionality is enabled and what remains outside the system.

### KISS design

Lunar favors simple, readable and transparent system behavior.

### Shell-script driven

Modules and their management system are based on shell scripts.

### Close to upstream

Software is kept close to upstream sources with minimal visible patching.

### Administrator focused

Lunar is made for users who want to understand and control Linux.
